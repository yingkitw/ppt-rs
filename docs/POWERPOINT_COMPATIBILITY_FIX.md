# PowerPoint Compatibility Fix

## Issue
PPTX files generated by ppt-rs could be read by python-pptx library (showing correct slide count) but could not be opened by Microsoft PowerPoint.

## Root Cause
The generated `presentation.xml` was missing critical elements required by PowerPoint:

### Missing Elements
1. **`<p:sldMasterIdLst>`** - Slide master list (CRITICAL)
2. **`saveSubsetFonts` and `autoCompressPictures` attributes** - Presentation attributes
3. **`type="screen4x3"` attribute** - Slide size type
4. **`<p:defaultTextStyle>`** - Default text styling for 9 paragraph levels

## Solution
Updated `PresentationPart::blob()` method in `/src/parts/presentation.rs` to generate complete presentation.xml matching python-pptx format.

### Changes Made

**Before:**
```xml
<?xml version="1.0" encoding="UTF-8" standalone="yes"?>
<p:presentation xmlns:a="..." xmlns:r="..." xmlns:p="...">
  <p:sldIdLst>
    <p:sldId id="256" r:id="rId1"/>
  </p:sldIdLst>
  <p:sldSz cx="9144000" cy="6858000"/>
  <p:notesSz cx="6858000" cy="9144000"/>
</p:presentation>
```

**After:**
```xml
<?xml version="1.0" encoding="UTF-8" standalone="yes"?>
<p:presentation xmlns:a="..." xmlns:r="..." xmlns:p="..."
                saveSubsetFonts="1" autoCompressPictures="0">
  <p:sldMasterIdLst>
    <p:sldMasterId id="2147483648" r:id="rId1"/>
  </p:sldMasterIdLst>
  <p:sldIdLst>
    <p:sldId id="256" r:id="rId1"/>
  </p:sldIdLst>
  <p:sldSz cx="9144000" cy="6858000" type="screen4x3"/>
  <p:notesSz cx="6858000" cy="9144000"/>
  <p:defaultTextStyle>
    <a:defPPr><a:defRPr lang="en-US"/></a:defPPr>
    <a:lvl1pPr marL="0" algn="l" defTabSz="457200" rtl="0" eaLnBrk="1" latinLnBrk="0" hangingPunct="1">
      <a:defRPr sz="1800" kern="1200">
        <a:solidFill><a:schemeClr val="tx1"/></a:solidFill>
        <a:latin typeface="+mn-lt"/><a:ea typeface="+mn-ea"/><a:cs typeface="+mn-cs"/>
      </a:defRPr>
    </a:lvl1pPr>
    <!-- ... levels 2-9 ... -->
  </p:defaultTextStyle>
</p:presentation>
```

## Key Additions

### 1. Slide Master Reference
```xml
<p:sldMasterIdLst>
  <p:sldMasterId id="2147483648" r:id="rId1"/>
</p:sldMasterIdLst>
```
- **Critical**: PowerPoint requires at least one slide master
- ID `2147483648` is the standard starting ID for slide masters
- `rId1` references the slide master relationship

### 2. Presentation Attributes
```xml
saveSubsetFonts="1" autoCompressPictures="0"
```
- `saveSubsetFonts="1"`: Embed only used font characters
- `autoCompressPictures="0"`: Don't auto-compress images

### 3. Slide Size Type
```xml
<p:sldSz cx="9144000" cy="6858000" type="screen4x3"/>
```
- Added `type="screen4x3"` attribute
- Specifies the aspect ratio of slides

### 4. Default Text Style
```xml
<p:defaultTextStyle>
  <a:defPPr><a:defRPr lang="en-US"/></a:defPPr>
  <a:lvl1pPr marL="0" algn="l" defTabSz="457200" ...>
    <a:defRPr sz="1800" kern="1200">
      <a:solidFill><a:schemeClr val="tx1"/></a:solidFill>
      <a:latin typeface="+mn-lt"/><a:ea typeface="+mn-ea"/><a:cs typeface="+mn-cs"/>
    </a:defRPr>
  </a:lvl1pPr>
  <!-- Levels 2-9 with increasing margins -->
</p:defaultTextStyle>
```
- Defines default text formatting for all 9 paragraph levels
- Each level has margin, alignment, tab size, and font properties
- Required for proper text rendering in PowerPoint

## Additional Fix: Duplicate Relationships

### Issue #2: Duplicate XML in Relationships File
After the first fix, files still couldn't open due to duplicate XML declarations in `ppt/_rels/presentation.xml.rels`.

**Root Cause:**
- Slide relationships were being added twice:
  1. In `Presentation::add_slide()` method
  2. In `save()` process via `PresentationRelationshipManager`
- This created duplicate `<?xml version...?>` declarations

**Solution:**
1. Remove relationship addition from `add_slide()` - only track in `SlideIdManager`
2. Remove manual `PresentationRelationshipManager` - let `PackageWriter` handle it
3. Add core relationships (rId1-rId6) in save process before slide relationships
4. Clear existing relationships before adding new ones

### Final Structure

**Presentation Relationships (rId1-rId10):**
- rId1: Slide Master
- rId2: Printer Settings
- rId3: Presentation Properties
- rId4: View Properties
- rId5: Theme
- rId6: Table Styles
- rId7-rId10: Slides 1-4

### Files Modified (Second Fix)
- `/src/presentation/presentation.rs` - Removed relationship addition from `add_slide()`
- `/src/presentation/save.rs` - Added core relationships, removed duplicate manager

## Additional Fix: Missing Leading Slash in PartNames

### Issue #3: PartNames Without Leading Slash
After fixing duplicates, files still failed with "content type is 'application/xml'" error.

**Root Cause:**
- [Content_Types].xml had PartNames without leading slash
- Our code: `<Override PartName="ppt/presentation.xml" ...>`
- Required: `<Override PartName="/ppt/presentation.xml" ...>`
- Without leading slash, python-pptx couldn't match Override entries

**Solution:**
Changed `PackageWriter` in `/src/opc/serialized.rs` to use `part.uri().as_str()` instead of `part.uri().membername()` for PartName attribute.

### Files Modified (Third Fix)
- `/src/opc/serialized.rs` - Use as_str() to include leading slash

## Additional Fix: Missing Transform Information

### Issue #4: Empty grpSpPr Elements
PowerPoint prompted to repair files due to missing transform information.

**Root Cause:**
- Slides had empty `<p:grpSpPr/>` elements
- PowerPoint expects transform data even for blank slides

**Solution:**
Updated slide XML generation to include:
```xml
<p:grpSpPr>
  <a:xfrm>
    <a:off x="0" y="0"/>
    <a:ext cx="0" cy="0"/>
    <a:chOff x="0" y="0"/>
    <a:chExt cx="0" cy="0"/>
  </a:xfrm>
</p:grpSpPr>
```

### Files Modified (Fourth Fix)
- `/src/presentation/save.rs` - Added transform information to slide XML
